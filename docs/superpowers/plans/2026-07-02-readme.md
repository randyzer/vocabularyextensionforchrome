````markdown
# 语境生词本

一个本地优先的 Chrome / Arc 英语阅读扩展：悬停查词、自动保存原句，并生成每周复习摘要。

当前版本：`0.1.0`  
当前状态：早期测试版，通过开发者模式手动安装，尚未发布到 Chrome Web Store。

## 它能做什么

- 鼠标在英文单词上稳定停留 500ms 后显示本地词典释义。
- 开启自动发音后，查词成功时调用浏览器的英语语音朗读。
- 查词成功并继续停留 2000ms 后，自动保存单词和所在句子。
- 同一页面的同一文本位置只保存一次，减少反复悬停造成的重复记录。
- 保存后 5 秒内可以撤销。
- 在侧边栏查看本期记录、生词库和历史周报。
- 按周生成本地复习摘要，并通过 Chrome 通知提醒。
- 支持导出、导入和清空本地数据。

扩展会忽略输入框、文本框、代码块和可编辑区域，避免读取正在输入的内容。

## 安装

Chrome 和 Arc 不能直接加载 ZIP 文件，必须先解压。

1. 获取 `contextual-vocabulary-0.1.0-chrome.zip`，解压到一个不会被随意移动或删除的固定目录。
2. 打开扩展管理页：
   - Chrome：`chrome://extensions`
   - Arc：`arc://extensions`；如果无法打开，使用 `chrome://extensions`
3. 开启右上角的“开发者模式”。
4. 点击“加载已解压的扩展程序”。
5. 选择解压后包含 `manifest.json` 的目录。
6. 点击浏览器工具栏中的扩展图标，打开“语境生词本”侧边栏。
7. 点击“允许在英语网页上取词”，授予网页访问权限。

扩展只会在获得授权后读取网页中的可见英文文本。

## 使用方法

### 查词和保存

1. 打开一篇英文网页。
2. 将鼠标停在一个英文单词上。
3. 保持 500ms，查看释义；开启自动发音时会同时朗读。
4. 继续停留 2000ms，单词和原句会自动保存。
5. 保存后如需取消，点击提示卡中的“已保存 · 撤销”。

鼠标移到其他位置、滚动页面或切换窗口，会取消尚未完成的查词或保存倒计时。

### 查看记录

点击扩展图标打开侧边栏：

- **本期**：查看尚未进入周报的近期记录。
- **生词库**：按词条查看累计遇见次数和原句。
- **周报**：查看每周生成的复习摘要。
- **设置**：控制扩展开关、自动发音、来源保存、通知时间和数据管理。

### 数据导入与导出

在侧边栏的“设置”中：

- 点击“导出 JSON”备份数据。
- 选择 JSON 文件恢复数据。
- 点击“清空全部数据”删除本地记录。

## 升级

为了尽量保留原扩展 ID 和本地数据：

1. 不要先删除浏览器中已安装的扩展。
2. 解压新版本，并用新文件覆盖原来的扩展目录。
3. 打开扩展管理页。
4. 在“语境生词本”卡片上点击“重新加载”。

新版本会自动激活已经打开且获得授权的网页。若网页权限被撤销，请重新打开侧边栏并授权。

如果改用新的解压目录重新加载，浏览器可能生成新的扩展 ID；旧扩展的数据仍属于旧 ID，因此看起来像“数据消失”。建议升级前先导出 JSON 备份。

## 常见问题

### 悬停没有反应

依次检查：

1. 扩展管理页中“语境生词本”是否已启用。
2. 侧边栏是否显示网页权限已授予。
3. 设置中的总开关是否开启，当前网站是否被停用。
4. 当前页面是否是普通的 `http://` 或 `https://` 网页。扩展不能运行在 `chrome://`、浏览器商店和其他受保护页面。
5. 鼠标是否停在输入框、代码块或可编辑区域；这些位置会被主动忽略。

### 没有发音

1. 在“设置”中确认“自动发音”已开启。
2. 确认系统或浏览器提供可用的英语语音。
3. 先检查是否能正常显示释义；只有查词成功后才会朗读。

### Chrome 不接受 ZIP

这是正常行为。请先解压 ZIP，再通过“加载已解压的扩展程序”选择包含 `manifest.json` 的目录。

### 更新后旧数据不可见

通常是因为从新目录加载后扩展 ID 发生变化。重新启用原扩展目录，导出 JSON，再在新扩展中导入。

## 隐私

语境生词本采用本地优先设计：

- 不读取密码框、表单输入、Cookie 或网页存储。
- 不保存完整网页或 HTML。
- 单词、句子、来源、设置和周报保存在本机扩展存储中。
- 不包含广告、分析或遥测。
- 不会把收藏内容或浏览活动发送给开发者或第三方服务器。

完整说明见 [PRIVACY.md](./PRIVACY.md)。

## 本地开发

### 环境

- Node.js 20 或更高版本
- pnpm
- Chrome 或 Chromium

### 获取源码

```bash
git clone https://github.com/randyzer/vocabularyextensionforchrome.git
cd vocabularyextensionforchrome
pnpm install
```

### 开发模式

```bash
pnpm dev
```

WXT 会生成开发版扩展。根据终端提示，在扩展管理页加载生成的目录。

### 测试和检查

```bash
pnpm test
pnpm typecheck
pnpm test:e2e
```

- `pnpm test`：运行 Vitest 单元和集成测试。
- `pnpm typecheck`：运行 TypeScript 严格类型检查。
- `pnpm test:e2e`：构建测试扩展并运行 Playwright 浏览器流程。

### 生产构建

```bash
pnpm build
```

构建结果位于：

```text
.output/chrome-mv3/
```

可以直接在扩展管理页加载该目录。

### 打包 ZIP

```bash
pnpm zip
```

默认安装包位于：

```text
.output/contextual-vocabulary-0.1.0-chrome.zip
```
````
